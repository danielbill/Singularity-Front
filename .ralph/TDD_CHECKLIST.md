# TDD 实现强制检查清单

**目的**: 防止跳过测试，确保遵循 TDD 流程

---

## 实现前检查

在开始编写任何实现代码之前，确认：

- [ ] 已运行 `/speckit.tasks` 生成 `tasks.md`
- [ ] 已阅读 `tasks.md` 中所有测试任务
- [ ] 测试文件已创建（如 `tests/test_xxx.py`）
- [ ] 第一个测试用例已编写（预期失败）

---

## 实现中检查

- [ ] 每个功能实现前，先编写对应测试
- [ ] 运行测试确认失败（RED）
- [ ] 编写最小实现使测试通过（GREEN）
- [ ] 运行所有测试确认无回归
- [ ] 重构代码（如有必要）
- [ ] 再次运行测试确认通过

---

## 实现后检查

- [ ] 所有单元测试通过
- [ ] 测试覆盖率 >= 80%
- [ ] 无 `console.log` 或调试代码残留
- [ ] 代码符合项目风格规范

## 快速命令

```bash

# 运行特定测试文件

# 查看测试覆盖率
pytest tests/ --cov=src --cov-report=term-missing
```
